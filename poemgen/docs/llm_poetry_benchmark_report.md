# LLM 古诗生成能力横评报告

> 基于 POEMetric 2025 框架 | 评测日期：2026-06-06

## 1. 评测概述

### 1.1 目标
评估主流大语言模型在 **七言绝句** 生成任务上的能力，涵盖格律、押韵、平仄、主题贴合度、词汇多样性等维度。

### 1.2 参评模型

| 厂商 | 模型 | 备注 |
|------|------|------|
| Anthropic | Claude Opus 4.6 | 旗舰 thinking 模型 |
| OpenAI | gpt-5.5 | 最新旗舰 |
| DeepSeek | DeepSeek-V4-Pro | 国产旗舰推理模型 |
| 智谱 | GLM-5.1 | 深度推理模型（部分超时） |
| MiniMax | MiniMax-M3 | ❌ 全部超时，未完成评测 |
| Moonshot | Kimi-K2.6 | ❌ 全部超时，未完成评测 |

> ⚠️ MiniMax-M3 和 Kimi-K2.6 因深度推理耗时超过 120s/请求，在 OneAPI 网关限制下全部超时。GLM-5.1 成功 5/10 首。

### 1.3 评测主题（10 个）
春日登高远眺、月下思乡、秋雨中的离别、雪夜独酌、江南水乡晨曦、边塞将士戍守、归隐山林、咏梅、中秋望月怀远、暮春送友远行

### 1.4 共用 Prompt
```
你是一位精通中国古典诗词的诗人。请严格按照要求创作七言绝句。

要求：
1. 格式：七言绝句（四句，每句恰好七个汉字，共28字）
2. 押韵：偶数句（第2、4句）末字须押韵，首句末字可押可不押
3. 平仄：尽量遵循七言绝句的平仄格律
4. 质量：语言凝练优美，意象生动，有诗意和意境
5. 原创：不得照搬或改写已有名句

输出格式：仅输出四句诗，每句一行，不加标点符号，不加任何解释、标题或注释。
```

## 2. 评估维度（POEMetric 2025 适配）

| 维度 | 方法 | 说明 |
|------|------|------|
| **格式正确率** | 规则检测 | 是否严格四句七字 |
| **押韵率** | 韵母宽松匹配 | 2/4句末字韵母归类是否相同 |
| **平仄合规率** | 七言四式匹配 | 与标准平仄句式的最佳匹配度 |
| **主题相关度** | 字符匹配 | query 字在诗中出现比例 |
| **词汇多样性(MATTR)** | Moving Average TTR | 字级滑动窗口类型-标记比 |
| **重复率** | bigram 统计 | 行内 2-gram 重复比例 |

## 5. 分析与发现

### 5.1 格式遵循
所有模型均达到 **100% 格式正确率**，说明当前旗舰 LLM 在严格格式约束下表现优异。

### 5.2 押韵能力
- **Claude Opus 4.6** 和 **GLM-5.1** 达到 100% 押韵率
- **gpt-5.5** 90%（1 首未押韵）
- **DeepSeek-V4-Pro** 70%（3 首未严格押韵）

### 5.3 平仄格律
- **GLM-5.1** 平仄合规 93%，表现最优，可能与其深度推理（reasoning）能力相关
- 其余三家在 89-91% 区间，差异不显著

### 5.4 主题相关度
所有模型主题得分偏低（28-35%），这是因为评估方法基于字符匹配——古诗讲究 **意在言外**，好诗往往不直接用题目中的字，而是通过意象传达主题。此指标更适合作为下限参考。

### 5.5 词汇多样性
所有模型 MATTR 均 ≥ 0.99，说明无明显用词重复，七言绝句 28 字的短格式天然保证了高多样性。

### 5.6 可用性与延迟
| 模型 | 成功率 | 平均响应时间 |
|------|:------:|:------:|
| Claude Opus 4.6 | 10/10 (100%) | ~15s |
| gpt-5.5 | 10/10 (100%) | ~3s |
| DeepSeek-V4-Pro | 10/10 (100%) | ~8s |
| GLM-5.1 | 5/10 (50%) | ~30-60s |
| MiniMax-M3 | 0/10 (0%) | >120s 超时 |
| Kimi-K2.6 | 0/10 (0%) | >120s 超时 |

---

## 6. 结论

1. **格式遵循不再是区分度**：所有旗舰模型均能完美遵守七言绝句的格式约束
2. **押韵是关键差异点**：Claude 和 GLM 表现最优，DeepSeek 相对较弱
3. **深度推理有利格律但牺牲速度**：GLM-5.1 平仄最佳，但延迟高、成功率低
4. **综合最优选择**：考虑质量+可用性，**Claude Opus 4.6** 是最均衡的选择（100%成功率 + 100%押韵 + 高平仄合规）
5. **性价比之选**：**gpt-5.5** 响应最快（~3s），质量接近，适合批量生成场景

## 8. 局限性与改进方向

1. **样本量有限**：每模型仅 10 首（GLM 仅 5 首），统计置信度有待提高
2. **主题评估过于简单**：字符匹配无法衡量意境传达，建议引入 LLM-as-judge
3. **缺少创意/情感维度**：POEMetric 完整框架包含创意、情感共鸣、修辞手法等高级维度，本次仅覆盖基础指标
4. **推理模型超时**：MiniMax-M3/Kimi-K2.6 未能参评，建议增加超时阈值或使用流式接口
5. **温度设置差异**：Kimi 仅支持 temperature=1，可能影响输出多样性的公平比较
